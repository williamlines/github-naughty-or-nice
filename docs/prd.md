## 1. Executive Summary

### 1.1 Product Vision

**Naughty or Nice** is a web application that analyzes a GitHub user's public activity for the current year and delivers a fun, data-driven "naughty or nice" verdict ‚Äî complete with category breakdowns, personalized AI-generated commentary, and shareable results.

### 1.2 Value Proposition

- **For developers:** A playful way to reflect on their GitHub habits and get actionable (but fun) feedback
- **For the community:** Shareable, viral content that sparks friendly competition
- **For the hackathon:** A delightful, holiday-themed project that showcases technical skill

### 1.3 Key Features (MVP)

| Feature               | Description                                                    |
| --------------------- | -------------------------------------------------------------- |
| **Username Input**    | Simple landing page to enter any GitHub username               |
| **Activity Analysis** | Analyze 6 categories of GitHub behavior against best practices |
| **Verdict System**    | 7-tier scoring from "Extremely Nice" to "Extremely Naughty"    |
| **AI Summary**        | Personalized Santa verdict generated by GPT-4o-mini            |
| **Share Cards**       | Social-optimized images for X/Twitter sharing                  |

---

## 2. Problem Statement

### 2.1 The Opportunity

Developers love to reflect on their work and compare themselves (playfully) to peers. End-of-year "wrapped" experiences have become cultural moments. GitHub provides rich public data, but there's no fun, shareable way to analyze it.

### 2.2 Why Now?

- **Holiday timing:** Christmas theme creates urgency and relevance
- **Hackathon format:** Condensed timeline forces MVP focus
- **Social virality:** Developers love sharing stats and achievements

### 2.3 What Success Looks Like

A working application that:

1. Analyzes any public GitHub user in <10 seconds
2. Delivers an entertaining, personalized verdict
3. Generates shareable images users actually post

### 5.2 Screen Specifications

#### Screen 1: Landing Page

**Purpose:** Capture username, set festive tone

| Element          | Specification                                                              |
| ---------------- | -------------------------------------------------------------------------- |
| **Headline**     | "Have you been Naughty or Nice on GitHub?"                                 |
| **Subhead**      | "Enter your username to find out if you're getting code or coal this year" |
| **Input**        | Single text field, GitHub username validation                              |
| **CTA**          | "Check My List" button                                                     |
| **Atmosphere**   | Festive, playful, minimal clutter                                          |
| **Nice-to-have** | Subtle snowfall animation, avatar preview on valid username                |

**Acceptance Criteria:**

- [ ] Username field validates format (alphanumeric + hyphens)
- [ ] Submit button disabled until valid username entered
- [ ] Pressing Enter submits the form
- [ ] Invalid username shows inline error message

#### Screen 2: Loading State

**Purpose:** Build anticipation, mask API latency

| Element       | Specification                                                                            |
| ------------- | ---------------------------------------------------------------------------------------- |
| **Animation** | Progress indicator with festive theming                                                  |
| **Messages**  | Rotating: "Checking your commits...", "Reviewing your PRs...", "Consulting the elves..." |
| **Duration**  | Minimum 2 seconds (theatrical delay) + actual API time                                   |

**Acceptance Criteria:**

- [ ] Shows immediately after form submit
- [ ] Messages rotate every 2 seconds
- [ ] Gracefully handles if API is faster than animation
- [ ] Shows for minimum 2 seconds even if API is instant

#### Screen 3: Results Page

**Purpose:** Deliver the verdict, encourage sharing

| Section                | Content                                        |
| ---------------------- | ---------------------------------------------- |
| **Header**             | User avatar, @username, profile link           |
| **Verdict Card**       | Tier emoji, label, score (0-100), flavor text  |
| **AI Summary**         | 2-3 sentence personalized Santa verdict        |
| **Category Breakdown** | 6 categories with scores, progress bars, quips |
| **Actions**            | Share to X, Copy Link, Check Another User      |

**Acceptance Criteria:**

- [ ] All 6 category scores displayed with progress bars
- [ ] Each category shows contextual quip based on score
- [ ] AI summary is unique and references actual stats
- [ ] Share buttons generate correct URLs/text
- [ ] "Check Another User" returns to landing page

#### Screen 4: Error States

| Error                  | Message                                                         | Tone       |
| ---------------------- | --------------------------------------------------------------- | ---------- |
| **User not found**     | "Hmm, Santa can't find that username. Are you sure they exist?" | Helpful    |
| **No public activity** | "This account has been very quiet... suspiciously quiet. ü§î"    | Playful    |
| **Rate limited**       | "The elves are overwhelmed! Try again in a few minutes."        | Apologetic |
| **API down**           | "GitHub's workshop is closed for maintenance. Check back soon!" | Reassuring |

**Acceptance Criteria:**

- [ ] Each error type shows appropriate message
- [ ] Errors include option to try again
- [ ] Rate limit errors show retry timing if available

### 5.3 Reveal Animation

The results page features a **dramatic envelope reveal**:

1. Envelope slides in from bottom
2. Brief pause for anticipation (0.5s)
3. Envelope opens with 3D flip effect
4. Verdict card emerges and scales to full size
5. Category scores animate in sequentially

**Acceptance Criteria:**

- [ ] Animation is smooth (60fps target)
- [ ] Total animation duration: 2-3 seconds
- [ ] Animation can be skipped by clicking/tapping
- [ ] Animation works on mobile devices

## 10. Dependencies & Risks

### 10.1 External Dependencies

| Dependency          | Purpose               | Risk Level | Mitigation                        |

| Risk                     | Probability | Impact | Mitigation                     |
| ------------------------ | ----------- | ------ | ------------------------------ |

| **OpenAI latency**       | Low         | Medium | Set timeout, use fallback      |
| **Scope creep**          | High        | High   | Strict MVP definition, say no  |
| **Animation complexity** | Medium      | Medium | Use Framer Motion, keep simple |
| **Large user data**      | Low         | Medium | Limit repos analyzed to 10     |

### 10.3 API Limitations

| Limitation                 | Impact                            | Handling                         |
| -------------------------- | --------------------------------- | -------------------------------- |
| Events API: 90 days max    | Can't see full year for some data | Show data completeness warning   |
| Events API: 300 events max | Heavy users lose old data         | Use commits/PR APIs for accuracy |
| Search API: 30 req/min     | Stricter rate limit               | Cache results, minimize calls    |
